# Maskify-TS

**Advanced data masking utility for Node.js & TypeScript** ‚Äî safely mask emails, phones, credit cards, IPs, JWTs, and deeply nested object fields with wildcard support.

[![npm version](https://img.shields.io/npm/v/maskify-ts.svg)](https://www.npmjs.com/package/maskify-ts)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Maskify is a lightweight, zero-dependency (core) utility designed for logging, analytics, and compliance scenarios (e.g., GDPR/PII redaction) where sensitive data must be obscured.

## ‚ö°Ô∏è Features

- ‚úÖ **Deep Masking:** Mask strings, objects, arrays, and nested fields with zero mutation (non-destructive).
- ‚úÖ **Class Decorators:** Declarative masking using `@Mask` on DTOs and Entities.
- ‚úÖ **Wildcard Support:** Easy traversal using dot-paths (`users[*].email`) and wildcards (`*`).
- ‚úÖ **Stream Support:** High-performance masking for large files and logs via Transform Streams.
- ‚úÖ **Advanced Modes:** Support for **Allowlist** (Mask everything *except* X) and **Blocklist**.
- ‚úÖ **Deterministic Masking:** Generate consistent hashes for analytics (count unique users without storing PII).
- ‚úÖ **Framework Ready:** Built-in middleware for **Express** and **Fastify**.
- ‚úÖ **Specialized Maskers:** Auto-detects Email, Phone, Credit Card, IPv4/IPv6, JWT, and URLs.
- ‚úÖ **CLI Tool:** Pipe logs directly from the command line.

üöÄ Quickstart

1. Basic Usage

Example :
import { Maskify } from 'maskify-ts';

// üìß Email
Maskify.mask('john.doe@example.com', { type: 'email' });
// Output: "jo****@e****.com"

// üí≥ Credit Card
Maskify.mask('4111111111111111', { type: 'card' });
// Output: "4111 **** **** 1111"

// üåê IP Address
Maskify.mask('192.168.1.50', { type: 'ip' });
// Output: "192.168.1.***"

// üîë JWT (Keeps header, masks payload/sig)
Maskify.mask('eyJhbGciOi...', { type: 'jwt' });
// Output: "eyJhbGciOi....********.********"

2. Class Decorators (TypeScript)

Ideal for NestJS, TypeORM, or standardized DTOs.

Example :
import { Mask, Maskify } from 'maskify-ts';

class UserDTO {
  @Mask({ type: 'email' })
  email: string;

  @Mask({ type: 'phone', maskChar: '#' })
  phone: string;

  // No decorator = No masking
  username: string;

  constructor(email: string, phone: string, username: string) {
    this.email = email;
    this.phone = phone;
    this.username = username;
  }
}

const user = new UserDTO('john@doe.com', '+1234567890', 'johndoe');
const masked = Maskify.maskClass(user);

console.log(masked);
// UserDTO { email: 'jo**@d**.com', phone: '+123#######90', username: 'johndoe' }

3. Object & Array Masking

Use wildcard syntax (*) to target fields in arrays or nested objects.

Example :
const users = [
  {
    id: 1,
    profile: { email: 'admin@company.com', phone: '+15550001111' },
    sessions: [{ token: 'sensitive-jwt-token' }]
  }
];

const masked = Maskify.maskSensitiveFields(users, {
  '[*].profile.email': { type: 'email' },
  '[*].sessions[*].token': { type: 'jwt' }
});

üõ°Ô∏è Advanced Strategies

Allowlist Mode (Strict Security)

Instead of specifying what to hide, specify what to keep. Everything else is masked.

Example :
const sensitiveData = {
  id: 123,
  timestamp: 1600000000,
  user: { name: 'John', secret: 'xyz' }
};

// Only 'id' and 'timestamp' will remain visible.
// Everything else defaults to generic masking.
const safe = Maskify.maskSensitiveFields(sensitiveData, {
  'id': {},
  'timestamp': {}
}, { mode: 'allow' });

Deterministic Masking (Analytics)

Generate consistent masks. Same input + same secret = same output. Useful for counting unique users in logs without revealing identities.

Example :
const email = 'user@gmail.com';
const opts = { secret: 'my-app-super-secret' };

const hash1 = Maskify.deterministic(email, opts);
const hash2 = Maskify.deterministic(email, opts);

console.log(hash1 === hash2); // true (e.g., "a3f12b9...")

üåä Streaming (High Performance)

For processing large log files (GBs) without memory issues, use MaskifyStream.

Example :
import { createReadStream, createWriteStream } from 'fs';
import { MaskifyStream } from 'maskify-ts/stream';

const read = createReadStream('production.log');
const write = createWriteStream('clean.log');

const maskStream = new MaskifyStream({
  'user.email': { type: 'email' },
  'context.ip': { type: 'ip' }
});

read.pipe(maskStream).pipe(write);

üåê Middleware Support

Fastify

Example :
import Fastify from 'fastify';
import { Maskify } from 'maskify-ts';

const app = Fastify();

// Register plugin
Maskify.use(app, {
  fields: ['email', 'password'],
  maskOptions: { maskChar: '*' }
}, 'fastify');

app.get('/', async () => ({ email: 'test@test.com', password: '123' }));
// Response: { "email": "te**@t**.com", "password": "***" }

Express

Example :
import express from 'express';
import { Maskify } from 'maskify-ts';

const app = express();

// Register middleware
app.use(Maskify.middlewares.express({
  fields: [
    'email',
    { name: 'phone', options: { visibleEnd: 2 } }
  ]
}));

üíª CLI Usage

Maskify includes a binary for piping logs directly from the terminal.

Example :
# Basic: Mask specific fields
cat production.log | npx maskify-ts -f "email,userId"

# Allowlist: Hide everything EXCEPT timestamp and level
cat production.log | npx maskify-ts --allow -f "timestamp,level"

# Custom Char: Use '#' instead of '*'
echo '{"pass":"12345"}' | npx maskify-ts -f "pass" -c "#"

‚öôÔ∏è Configuration

MaskOptions Interface

Example :
interface MaskOptions {
  type?: 'email' | 'phone' | 'card' | 'ip' | 'jwt' | 'url' | 'generic';
  visibleStart?: number; // Chars visible at start
  visibleEnd?: number;   // Chars visible at end
  maxAsterisks?: number; // Max length of mask string
  maskChar?: string;     // Default: '*'
  autoDetect?: boolean;  // Default: true
  pattern?: string;      // e.g. "###-**-###"
  transform?: (val: string) => string; // Custom function
}
